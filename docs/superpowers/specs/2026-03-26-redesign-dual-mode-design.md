# IstiLik — Redesign & Dual Mode Feature Spec

**Date:** 2026-03-26
**Status:** Approved
**Approach:** B — "Два режима + Новый дизайн"

---

## 1. Контекст и цели

IstiLik — Android приложение для расчёта отопления (секции радиаторов, мощность котла), ориентированное на Азербайджан. Единственное подобное приложение на азербайджанском языке на Google Play.

**Цели редизайна:**
- Современный визуальный стиль (тёмная тема с акцентными цветами)
- Два режима использования: простой (обычные пользователи) и расширенный (сантехники/монтажники)
- Улучшенный UX — меньше полей видно сразу, живые результаты
- Основа для будущего Подхода C (полная платформа с проектами и базой оборудования)

---

## 2. Аудитория

- **Обычные пользователи** — владельцы квартир/домов, хотят быстро прикинуть мощность котла. Приоритет: простота, понятность, минимум полей.
- **Профессионалы** — сантехники и монтажники отопления. Приоритет: точность расчёта, все параметры доступны, PDF отчёт для клиента.

## 3. Визуальный стиль

### Цветовая схема (тёмная тема — основная)

| Роль | Цвет | Hex |
|------|------|-----|
| Фон приложения | Почти чёрный | `#0F0F14` |
| Поверхность карточек | Тёмно-синий | `#1A1A24` |
| Акцент (кнопки, активные элементы) | Оранжево-красный | `#FF6B35` |
| Вторичный акцент (иконки, подсветка) | Янтарный | `#FFB347` |
| Основной текст | Белый | `#FFFFFF` |
| Вторичный текст | Серо-синий | `#8A8A9A` |
| Ошибка | Красный | Material error color |

### Типографика
- Результат расчёта — самый крупный элемент на экране (32sp+, жирный)
- Заголовки карточек — средние, полужирные
- Подписи полей — мелкие, вторичный цвет

### Карточки
- Скруглённые углы (12dp)
- Мягкая тень или лёгкая граница
- Акцентные элементы с градиентной границей

## 4. Структура экранов

### Нижняя навигация
2 вкладки:
- 🔥 **Расчёт** — главный экран
- 📋 **История** — история расчётов

### Главный экран (вкладка "Расчёт")

```
Шапка
  └── Название приложения + иконка пламени + кнопка смены языка

Карточка "Помещение"
  ├── Площадь (м²)  [TextInputEditText]
  └── Высота потолков (м)  [TextInputEditText]

Карточка "Параметры"
  ├── Тип жилья: [Квартира | Дом]  — сегментированная кнопка
  └── Тип системы: [Радиаторы | Тёплый пол | Смешанный]  — сегментированная кнопка

Кнопка "Расширенные параметры ▼"  (скрыта по умолчанию, раскрывается с анимацией)

  Карточка "Конструкция"  [скрыта по умолчанию]
    ├── Кол-во внешних стен  [дропдаун]
    ├── Утепление  [дропдаун: Хорошее / Среднее / Плохое]
    └── Тип перекрытия  [дропдаун: Стандарт / Чердак]

  Карточка "Радиатор и ГВС"  [скрыта по умолчанию]
    ├── Мощность секции радиатора (Вт)  [TextInputEditText]
    ├── Площадь тёплого пола (м²)  [TextInputEditText]
    └── Точки ГВС  [дропдаун: 1–9]

Блок результата (всегда виден, прикреплён над кнопками)
  ├── Требуемая мощность: XX.X кВт  [крупный текст, акцентный цвет]
  ├── Секций радиатора: XX
  └── Рекомендуемый котёл: XX кВт

Кнопки действий
  ├── [Сохранить в историю]  (основная, акцентная)
  ├── [Сбросить]
  └── [PDF отчёт]
```

### Экран "История"
- Список карточек: дата + площадь + мощность котла
- Свайп влево → удалить запись
- Нажатие на карточку → восстанавливает параметры в форму
- Кнопка "Очистить всё" в меню

## 5. UX поведение

### Живые результаты
- Блок результата обновляется при каждом изменении поля
- Кнопка "Сохранить в историю" добавляет текущий расчёт в историю
- Пока поля не заполнены корректно — показывается `— кВт`, `— секций`

### Расширенные параметры
- Скрыты по умолчанию — при входе видно только 4 поля
- Кнопка с иконкой стрелки раскрывает секцию с плавной анимацией expand
- Состояние (открыто/закрыто) сохраняется в SharedPreferences

### Переключатели
- "Квартира / Дом" — `MaterialButtonToggleGroup` (сегментированная кнопка)
- "Радиаторы / Тёплый пол / Смешанный" — то же самое
- Заменяют дропдауны для этих двух параметров (быстрее, современнее)

### Валидация
- Ошибки показываются только после того как пользователь коснулся поля и вышел
- Ошибочное поле подсвечивается красным через `TextInputLayout.setError()`
- При `UNDERFLOOR_ONLY` поле мощности радиатора скрывается полностью (не только disabled)

### Анимации
- Результат при изменении: лёгкий fade (150ms)
- Раскрытие расширенных параметров: плавный expand (250ms)

## 6. Технические изменения

### Что НЕ меняется
- `HeatingCalculator.java` — вся логика расчёта сохраняется полностью
- SharedPreferences как хранилище состояния
- FileProvider + PDF генерация
- Все 3 языка (az/ru/en)
- `minSdk = 24`, `targetSdk = 36`

### Рефакторинг MainActivity.java (730 строк → разбивка)
- `HistoryManager.java` — чтение/запись/удаление истории из SharedPreferences
- `PreferencesHelper.java` — сохранение/восстановление состояния формы
- `MainActivity.java` — только UI логика и координация

### Новые/изменённые layout файлы
- `activity_main.xml` — полный редизайн с карточками и BottomNavigationView
- `fragment_calculator.xml` — экран расчёта (Fragment)
- `fragment_history.xml` — экран истории (Fragment)
- `item_history_card.xml` — карточка записи в истории

### Ресурсы
- `themes.xml` — тёмная тема как основная (`Theme.IstiLik.Dark`)
- `colors.xml` — новая палитра (`#0F0F14`, `#1A1A24`, `#FF6B35`, `#FFB347`)
- Новые строки: "Расширенные параметры", "Скрыть параметры", "Сохранить в историю" во всех 3 языках

### Навигация
- `BottomNavigationView` с 2 вкладками
- `Fragment` для каждой вкладки
- Без новых зависимостей — только Material Design 3 (уже есть в проекте)
